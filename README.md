# Task Capsule

大学生向けの課題管理Webアプリです。課題の締切日時、所要時間、メモを管理し、タスクの処理結果をタスクボール・コップ・キャラクター成長として可視化します。

## 概要

Task Capsuleは、大学のレポート、試験勉強、提出物などをブラウザだけで管理できるタスク管理アプリです。サーバーやデータベースは使用せず、登録内容はブラウザのlocalStorageへ保存します。

## 目的

- 課題と締切を分かりやすく整理する
- 締切超過タスクを見落としにくくする
- タスク完了をキャラクター成長へつなげ、継続する動機を作る
- 未完了の結果も可視化し、次の行動につなげる

## 主な機能

- ニックネームの登録・変更
- 課題名、締切日、締切時刻、所要時間、メモの登録
- 締切超過タスクの優先表示と赤枠表示
- 締切順、所要時間の長い順による並び替え
- 完了、未完了、削除
- タスクごとに固定された色のタスクボール
- JUICE・POISONコップの5段階表示
- コップ満杯時のキャラクターレベル変化
- レベルに応じたキャラクター画像切り替え
- localStorageによる状態保存・復元
- ニックネームを残したままタスクと成長状態を初期化するリセット

## 使用技術

- HTML
- CSS
- JavaScript
- localStorage
- 外部ライブラリなし

## ファイル構成

```text
TaskManager_10/
├─ index.html
├─ style.css
├─ script.js
├─ character-images.js
├─ character-image-loader.js
└─ README.md
```

キャラクター画像は`character-images.js`にPNGデータとして埋め込み、`character-image-loader.js`でレベルに応じた画像へ差し替えます。画像が読み込めない場合は、レベル番号を含む代替表示が使用されます。

## 使い方

1. `index.html`をブラウザで開きます。
2. 初回画面でニックネームを入力し、「保存して次へ」を押します。
3. 「＋ タスク追加」から課題を登録します。
4. タスク一覧で締切、所要時間、メモを確認します。
5. タスクの結果に応じて「完了」「未完了」「削除」を押します。
6. 「ホーム」からニックネームを変更できます。タスクや成長状態は保持されます。
7. 「リセット」では、確認後にタスク・コップ・レベルを初期化します。ニックネームは保持されます。

## タスクの状態変化

| 操作 | タスク一覧 | タスクボール | コップ |
| --- | --- | --- | --- |
| 完了 | 対象タスクを削除 | 対応する色のボールを削除 | JUICEを1増加 |
| 未完了 | 対象タスクを削除 | 対応する色のボールを削除 | POISONを1増加 |
| 削除 | 対象タスクを削除 | 対応する色のボールを削除 | 変化なし |

未完了ボタンは、締切日時を過ぎたタスクにだけ表示されます。

## タスクボール・コップ・キャラクターの仕組み

### タスクボール

- 未処理タスク1件につき1個表示します。
- 色はタスク登録時に決まり、タスクデータの`color`へ保存されます。
- 再描画や再読み込みでは色が変わりません。
- タスクIDを基準に処理されるため、完了・未完了・削除したタスクに対応するボールだけが消えます。

### JUICE・POISON

- 各コップは5ブロックで満杯です。
- 完了するとJUICEが増えます。
- 締切超過タスクを未完了にするとPOISONが増えます。
- JUICEが満杯になるとキャラクターレベルが1上がり、JUICEは0に戻ります。
- POISONが満杯になるとキャラクターレベルが1下がり、POISONは0に戻ります。

### キャラクター

- 初期レベルは2です。
- レベルは1〜5の範囲です。
- レベル1〜5に応じて、埋め込み済みのキャラクター画像を表示します。
- リセット後はレベル2へ戻ります。

## データ保存

状態はlocalStorageの`taskAppState`キーへ保存されます。

保存対象：

- ニックネーム
- タスク一覧
- JUICEカウント
- POISONカウント
- キャラクターレベル
